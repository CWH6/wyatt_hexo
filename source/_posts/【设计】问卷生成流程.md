> 从 `/genOnline` 接口出发，梳理一次完整问卷 HTML 生成的链路：数据准备 → FreeMarker 渲染 → 文件落盘 → 链接入库。



## 💡 背景

问卷系统通过 FreeMarker 模板将项目数据（题目、面板、配置等）渲染为静态 HTML，一个接口 `/genOnline` 完成全部流程。主要涉及：

- 循环题数据展开（伪题变真题）
- 组合题父子关系处理
- 多语言国际化适配
- 版本号管理（自增 or 固定）
- HTML 压缩（2.7MB → ~1.2MB）



## 🏗️ 整体流程

```
POST /projects/genOnline
  │
  ▼
ProjectsController.genOnline(dto)
  │
  ▼
ProjectsManager.genOnline(dto)
  │
  ├─ ① validateInput(dto)            → 校验 projectId 非空
  ├─ ② validateAndSetParams(dto)     → 参数补全和校验
  ├─ ③ getOrInitVersion(key)         → 从 Redis 获取/初始化版本号
  ├─ ④ buildProjectData(dto)         → 构建 FreeMarker 数据模型
  │    ├─ selectProjectsById         → 查项目 + 题目 + 面板 + 设置
  │    ├─ processProjectsLanguage    → 多语言数据适配
  │    ├─ processProjectQuestions    → 组合题父子排序
  │    ├─ processLoopV2              → 循环题伪题展开为真子题
  │    └─ scanAndSetPlaceholderValues→ 占位符初值扫描
  │
  ├─ ⑤ generateMainHtml(data)         → FreeMarker 渲染 + 压缩
  │    ├─ initFreeMarkerConfig()     → 加载 /template 下 FTL
  │    ├─ cfg.getTemplate("html_online_frame.ftl")
  │    ├─ template.process(data, sw) → 模板渲染
  │    └─ compressHtml(sw)           → 去除标签间空白
  │
  ├─ ⑥ saveLink(dto, currVersion)    → 生成链接记录入库
  │    └─ projectsLinkService.saveOrUpdate()
  │
  ├─ ⑦ projectPathVersionUpdate()    → 版本号自增（非 fixed 模式）
  ├─ ⑧ clearOldData()               → 清理旧答卷数据（可选）
  ├─ ⑨ saveProjectCurrentVersionBackup() → 项目快照备份
  ├─ ⑩ genGuid()                     → 异步生成 GUID（如有勾选）
  │
  └─ return getResponsePath()        → 返回访问 URL
```



## 🔑 关键步骤详解

### 步骤 ④：数据准备 — `buildProjectData()`

这是整个流程中最核心的一步，为 FreeMarker 模板构建完整的 `Map<String, Object>` 数据模型：

```java
private Map<String, Object> buildProjectData(ProjectGenerationDTO dto, 
        String linkType, Long currVersion) {
    ProjectsVO projects = selectProjectsById(dto.getProjectId());
    
    // 1. 多语言处理：把 i18n 配置项按当前主语言展开到题目字段
    processProjectsLanguage(projects);
    
    // 2. 组合题处理：按 orderNum 数值排序 + 建立父子映射
    processProjectQuestions(projects);
    
    // 3. 循环题展开：伪题 L1 拆为 L1_b1_r1, L1_b1_r2... 真子题
    processLoopV2(projects);
    
    // 4. 扫描占位符初值（{{xxx}} 变量在题目中的默认值）
    Map<String, Object> placeholderValues = scanAndSetPlaceholderValues(projects);

    // 5. 构建 data Map
    Map<String, Object> data = new HashMap<>();
    data.put("project", projects);                // 完整项目对象
    data.put("projectName", projects.getName());
    data.put("isNav", dto.getIsNav());            // 是否显示导航栏
    data.put("isPre", dto.getIsPre());            // 是否预加载下一题
    data.put("isTest", ...);                      // 是否测试模式
    data.put("currVersion", currVersion);         // 当前版本号
    data.put("freemarketStaticBaseUrl", ...);     // JS/CSS 静态资源基础 URL
    data.put("i18nMode", projects.getI18nMode()); // 国际化模式
    data.put("primaryLang", projects.getPrimaryLang());
    // ... 其余控制参数（idCtrl, breakpointCtrl, record 等）
    return data;
}
```

**循环题展开** 的核心逻辑：

```
原始数据：
  L1（循环题：loopCount=2）
    ├── L1_b1（基础子题）
    └── L1_b2（基础子题）

展开后：
  L1_b1_r1  ← 第一次循环的子题1
  L1_b2_r1  ← 第一次循环的子题2
  L1_b1_r2  ← 第二次循环的子题1
  L1_b2_r2  ← 第二次循环的子题2

→ 循环题本身被移除，子题替代到原位置，orderNum 重新编号
```

### 步骤 ⑤：FreeMarker 渲染 — `generateMainHtml()`

```java
private String generateMainHtml(Map<String, Object> projectData,
        String filePath, String projectPath, ProjectGenerationDTO dto) {
    // 1. 初始化 FreeMarker，从 /template 目录加载模板
    Configuration cfg = new Configuration(VERSION_2_3_30);
    cfg.setClassForTemplateLoading(ProjectsManager.class, "/template");

    // 2. 选择模板（默认 default 主题，可选 luxury）
    Template template;
    if ("luxury".equals(dto.getTheme())) {
        template = cfg.getTemplate("/theme/luxury/html_online_frame.ftl");
    } else {
        template = cfg.getTemplate("html_online_frame.ftl");
    }

    // 3. 渲染到 String + 压缩
    StringWriter sw = new StringWriter();
    template.process(projectData, sw);
    String html = compressHtml(sw.toString());

    // 4. 写入文件系统
    File htmlFile = new File(convertPath(filePath + projectPath));
    htmlFile.getParentFile().mkdirs();
    try (Writer writer = new OutputStreamWriter(
            new FileOutputStream(htmlFilePath), StandardCharsets.UTF_8)) {
        writer.write(html);
    }
}
```

**HTML 压缩策略**：保留 `<script>` `<style>` `<pre>` `<textarea>` 内容不动，其余部分去除标签间空白 → 典型压缩率 40%-60%（2.7MB → ~1.2MB）。

### 步骤 ⑥：链接入库 — `saveLink()`

| 字段 | 来源 | 说明 |
|------|------|------|
| `project_id` | dto.projectId | 项目 ID |
| `file_path` | filePath + projectPath | 本地文件路径 |
| `link_url` | domainPath + "/" + projectPath | 对外访问 URL |
| `version` | currVersion | 版本号 |
| `type` | dto.type | 链接类型（test/prod1/prod2 等） |
| `tags` | JSON | 生成时的控制参数（ipCtrl/guiCtrl 等） |
| `is_enable` | 1 | 默认启用 |

同一 `projectId + version + type` 复用已有记录 ID（`saveOrUpdate`），不会重复创建。

### 版本号管理

| 模式 | Redis key | 行为 |
|------|-----------|------|
| 固定版本（`fixed=Y`） | 不递增 | 版本号不变，每次覆盖旧文件 |
| 自增版本（默认） | `survey:version:{projectId}:{type}` | 每次 +1，旧版本保留 |

输出文件路径格式：

```
filePath/type/projectId/version/index.html
→ /data/html/test/123/5/index.html          ← 测试链接第 5 版
→ /data/html/prod1/123/3/index.html         ← 正式链接第 3 版
```

访问 URL 格式：

```
domainPath/type/projectId/version/
→ https://v2.yisurvey.com/test/123/5/
```



## 📊 涉及的数据模型

| 模型 | 查询时机 | 用途 |
|------|----------|------|
| `projects` | `selectProjectsById` | 项目基本信息 |
| `projects_question` | 随 projects 联查 | 所有题目列表 |
| `projects_panel` | `panelService.getByProjectId` | 题板/分页信息 |
| `projects_setting` | `settingService.selectById` | 项目设置（皮肤、语言等） |
| `projects_link` | `saveLink` 时查询 | 判断是否已有记录 |



## 🎯 主题切换

当前支持两种主题，通过 `dto.theme` 控制：

| theme 值 | 模板 | 说明 |
|------|------|------|
| `luxury` | `/theme/luxury/html_online_frame.ftl` | 豪华版主题 |
| 其他/空 | `html_online_frame.ftl` | 默认主题 |

> 题型片段模板（如 `text.ftl`、`radio.ftl` 等）两套主题共用，主题差异只在主壳 `html_online_frame.ftl` 里体现。



## ⚡ 异步旁路

`genGuid()` 在生成完成后异步执行，不阻塞接口响应：

```java
if("N".equals(guidCtrl)) return;  // 没勾选 GUID 则跳过

// 未生成过 → 异步生成
CompletableFuture.runAsync(() -> {
    projectsRespondentService.genDefaultGuids(projectId);
});
redisCache.setCacheObject("guid:" + projectId, 1);  // 标记已生成
```



## ⭐ 设计要点总结

1. **一次请求完成全部**：单接口从数据准备到文件落盘到链接入库，无需多次调用
2. **循环题智能展开**：通过 `processLoopV2()` 将伪题按 `loopCount` 自动生成 N 批真子题
3. **版本号管理**：自增模式保留历史版本，固定模式覆盖复用
4. **HTML 压缩**：不影响脚本和样式的前提下压缩 40%+，减少首次加载体积
5. **项目快照备份**：每次生成自动备份到 `projects_backup` 表，方便回滚
6. **异步非阻塞**：GUID 生成、旧数据清理等非关键操作异步或可选执行
