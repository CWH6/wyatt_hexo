> 一套支持模板变量、个性化正文、定时发送、逐条追踪的问卷邮件推送系统。



## 💡 背景

问卷系统需要向答题者推送邀请邮件。核心需求：

- 支持多个 SMTP 发送人（不同项目用不同发件箱）
- 邮件正文支持模板 + 变量替换（姓名、链接等）
- 支持定时发送，到点自动推送
- 每条邮件发送结果可追踪（成功/失败/错误原因）
- 收件人可分组建



## 🏗️ 整体架构

```
┌──────────────────────────────────────────────┐
│                ProjectsEmailController        │  ← REST API
│   /email/sender  /template  /group  /user     │
│   /email/send/list  /send/detail  /send/execute│
└────────────────────┬─────────────────────────┘
                     │
┌────────────────────▼─────────────────────────┐
│              ProjectsEmailManager             │  ← 业务编排
│   增删改查 + 调用 EmailSendService 发邮件      │
└────────────────────┬─────────────────────────┘
                     │
┌────────────────────▼─────────────────────────┐
│              EmailSendService                 │  ← 核心发送引擎
│   resolveRecipients → createMailSession       │
│   → 逐个 sendMailDirect → 记录 detail          │
└────────────────────┬─────────────────────────┘
                     │
┌────────────────────▼─────────────────────────┐
│       EmailScheduleTask（每5分钟扫描）          │  ← 定时任务
│   查 status=4 且 scheduled_at ≤ now 的记录     │
│   → 调用 EmailSendService.executeExisting()   │
└──────────────────────────────────────────────┘
```

## 🔄 核心流程

### 流程一：立即发送

```
用户提交 SendEmailDTO
  → 校验发送人 SMTP 配置
  → 解析收件人列表（分组 or 手动选择）
  → 生成问卷链接（查 projects_link 表）
  → 构建模板变量（{{name}} {{email}} {{link}}）
  → 创建 send_record（status=1 发送中）
  → javax.mail 逐个发送
     ├─ 成功 → detail.status=1, 记录 sent_at
     └─ 失败 → detail.status=2, 记录 errorMsg
  → 更新 send_record（successCount / failCount / status）
```

### 流程二：定时发送

```
用户提交 scheduledAt 不为空的 SendEmailDTO
  → 预创建 send_record（status=4 等待定时）
  → 预创建 detail 快照（status=0，正文为空）
  → 返回"已预约"

定时任务 EmailScheduleTask（每 5 分钟）
  → 查 status=4 且 scheduled_at ≤ now 的记录
  → 依次调用 executeExisting()
     → 从 detail 表加载收件人快照
     → 逐个发送并更新 detail
     → 更新 send_record 最终状态
```

### 模板变量替换

```
模板：您好 {{name}}，请点击：{{link}}

替换逻辑：
  {{name}}  → recipient.name（收件人姓名）
  {{email}} → recipient.email（收件人邮箱）
  {{link}}  → 优先取 user.link_url（每人专属链接）
              其次取项目级问卷链接（查 projects_link 表）
              都没有则显示"还未生成问卷链接"
```



## 👥 收件人来源

两种方式，通过 `recipientSource` 区分：

| 来源 | 值 | 逻辑 |
|------|------|------|
| 按分组 | 1 | 从 `projects_email_user_group_rel` → `projects_email_user` 取出该组所有用户 |
| 手动选择 | 2 | 从 `userIds` 列表或 `emails` 列表直接获取 |

## 设计要点总结

1. **模板 + 变量**：通过 `{{name}}` `{{email}}` `{{link}}` 实现千人千面的个性化邮件，每人可配专属链接
2. **正文优先级**：模板 body_template > 用户 email_content，灵活覆盖
3. **定时 + 立即双模**：同一接口，`scheduledAt` 为空立即发，不为空预约发；定时任务 5 分钟扫一次
4. **逐条追踪**：每个收件人一条 detail，成功/失败/错误原因全部记录，出问题能精确到人
5. **SMTP 多租户**：不同项目配不同发件箱，互不干扰
6. **分组管理**：收件人通过多对多关联灵活分组，支持批量导入导出
