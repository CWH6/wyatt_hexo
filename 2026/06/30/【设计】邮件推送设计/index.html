【设计】邮件推送设计

发表于2026-06-30|更新于2026-07-04|设计后端
|浏览量:

一套支持模板变量、个性化正文、定时发送、逐条追踪的问卷邮件推送系统。

💡 背景

问卷系统需要向答题者推送邀请邮件。核心需求：

  • 支持多个 SMTP 发送人（不同项目用不同发件箱）
  • 邮件正文支持模板 + 变量替换（姓名、链接等）
  • 支持定时发送，到点自动推送
  • 每条邮件发送结果可追踪（成功/失败/错误原因）
  • 收件人可分组建

🏗️ 整体架构

┌──────────────────────────────────────────────┐
│                ProjectsEmailController        │  ← REST API
│   /email/sender  /template  /group  /user     │
│   /email/send/list  /send/detail  /send/execute│
└────────────────────┬─────────────────────────┘
                     │
┌────────────────────▼─────────────────────────┐
│              ProjectsEmailManager             │  ← 业务编排
│   增删改查 + 调用 EmailSendService 发邮件      │
└────────────────────┬─────────────────────────┘
                     │
┌────────────────────▼─────────────────────────┐
│              EmailSendService                 │  ← 核心发送引擎
│   resolveRecipients → createMailSession       │
│   → 逐个 sendMailDirect → 记录 detail          │
└────────────────────┬─────────────────────────┘
                     │
┌────────────────────▼─────────────────────────┐
│       EmailScheduleTask（每5分钟扫描）          │  ← 定时任务
│   查 status=4 且 scheduled_at ≤ now 的记录     │
│   → 调用 EmailSendService.executeExisting()   │
└──────────────────────────────────────────────┘

核心 5 表（配置层）

1. projects_email_sender — 发送人配置

字段 类型 说明

name varchar 发件人名称
smtp_host varchar SMTP 服务器地址
smtp_port int SMTP 端口
smtp_encryption int 加密方式：0-无、1-SSL、2-TLS
smtp_username varchar SMTP 认证用户名
smtp_password varchar SMTP 认证密码

2. projects_email_template — 邮件模板

字段 类型 说明
id bigint 主键
name varchar 模板名称
template_type int 模板类型
subject_template varchar 主题模板（支持变量）
body_template text 正文模板（支持变量）

支持的模板变量：{{name}}、{{email}}、{{link}}

3. projects_email_group — 收件人分组

字段 类型 说明
id bigint 主键
project_id bigint 所属项目
name varchar 分组名称
remark varchar 备注

4. projects_email_user — 收件人

字段 类型 说明
id bigint 主键
project_id bigint 所属项目
email varchar 邮箱地址
name varchar 收件人姓名
attributes varchar 自定义属性（JSON）
link_url varchar 每人专属问卷链接（可用于 ）
email_content text 每人自定义正文（优先级低于模板）
status int 状态

5. projects_email_user_group_rel — 用户-分组关联

字段 类型 说明
id bigint 主键
user_id bigint 用户 ID
group_id bigint 分组 ID

多对多关系，一个用户可属于多个组。

发送流水 2 表（记录层）

6. projects_email_send_record — 发送批次记录

字段 类型 说明

template_id bigint 使用的模板 ID
sender_id bigint 发件人 ID
send_type int 发送类型
recipient_source int 收件人来源：1-分组、2-手动选择
group_id bigint 分组 ID（来源为分组时）
subject varchar 实际发送的主题
total_count int 总收件人数
success_count int 成功数
fail_count int 失败数
status int 状态：1-发送中、2-已完成、3-全部失败、4-等待定时
scheduled_at datetime 预约发送时间
cron_expression varchar Cron 表达式（预留周期发送）
remark varchar 备注

7. projects_email_send_detail — 逐条发送明细

字段 类型 说明
id bigint 主键
send_id bigint 关联 send_record

email varchar 实际发送邮箱
name varchar 收件人名称
email_content text 实际发送的正文
status int 发送状态：0-未发送、1-成功、2-失败
error_msg varchar 失败原因
sent_at datetime 发送时间
opened_at datetime 邮件打开时间（预留追踪）
clicked_at datetime 链接点击时间（预留追踪）

🔄 核心流程

流程一：立即发送

用户提交 SendEmailDTO
  → 校验发送人 SMTP 配置
  → 解析收件人列表（分组 or 手动选择）
  → 生成问卷链接（查 projects_link 表）
  → 构建模板变量（{{name}} {{email}} {{link}}）
  → 创建 send_record（status=1 发送中）
  → javax.mail 逐个发送
     ├─ 成功 → detail.status=1, 记录 sent_at
     └─ 失败 → detail.status=2, 记录 errorMsg
  → 更新 send_record（successCount / failCount / status）

流程二：定时发送

用户提交 scheduledAt 不为空的 SendEmailDTO
  → 预创建 send_record（status=4 等待定时）
  → 预创建 detail 快照（status=0，正文为空）
  → 返回"已预约"

定时任务 EmailScheduleTask（每 5 分钟）
  → 查 status=4 且 scheduled_at ≤ now 的记录
  → 依次调用 executeExisting()
     → 从 detail 表加载收件人快照
     → 逐个发送并更新 detail
     → 更新 send_record 最终状态

模板变量替换

模板：您好 {{name}}，请点击：{{link}}

替换逻辑：
  {{name}}  → recipient.name（收件人姓名）
  {{email}} → recipient.email（收件人邮箱）
  {{link}}  → 优先取 user.link_url（每人专属链接）
              其次取项目级问卷链接（查 projects_link 表）
              都没有则显示"还未生成问卷链接"

👥 收件人来源

两种方式，通过 recipientSource 区分：

来源 值 逻辑
按分组 1 从 projects_email_user_group_rel → projects_email_user 取出该组所有用户
手动选择 2 从 userIds 列表或 emails 列表直接获取

API 接口清单

接口 方法 说明
/email/sender/list GET 发件人列表
/email/sender POST/PUT/DELETE 增改删发件人
/email/template/list GET 模板列表
/email/template POST/PUT/DELETE 增改删模板
/email/group/list GET 分组列表
/email/group POST/PUT/DELETE 增改删分组
/email/user/list GET 收件人列表
/email/user POST/PUT/DELETE 增改删收件人
/email/user/{userId}/group/{groupId} POST/DELETE 用户加入/移出分组
/email/user/import/{projectId} POST 批量导入收件人（Excel）
/email/user/export/{projectId} GET 导出收件人
/email/send/list GET 发送记录列表
/email/send/detail/{sendId} GET 查看某次发送的逐条明细
/email/send/execute POST 执行发送（核心接口）

设计要点总结

  1. 模板 + 变量：通过 {{name}} {{email}} {{link}} 实现千人千面的个性化邮件，每人可配专属链接
  2. 正文优先级：模板 body_template > 用户 email_content，灵活覆盖
  3. 定时 + 立即双模：同一接口，scheduledAt 为空立即发，不为空预约发；定时任务 5 分钟扫一次
  4. 逐条追踪：每个收件人一条 detail，成功/失败/错误原因全部记录，出问题能精确到人
  5. SMTP 多租户：不同项目配不同发件箱，互不干扰
  6. 分组管理：收件人通过多对多关联灵活分组，支持批量导入导出
